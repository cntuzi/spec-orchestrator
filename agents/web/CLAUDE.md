## Mandatory Rules

### 1. Build Verification
Every code change must pass:
```bash
{PACKAGE_MANAGER} run build
{PACKAGE_MANAGER} run typecheck
```
Expected output: no errors. Fix immediately on failure -- do not accumulate errors.

### 2. Build Loop
```
modify -> build + typecheck -> fix errors -> repeat until clean
```
Never skip the build step. Never proceed to the next task with a broken build.

### 3. Lint Check
```bash
{PACKAGE_MANAGER} run lint
```
Fix lint warnings before committing. Use `--fix` for auto-fixable issues.

### 4. Foreground Execution
- Run all commands in the foreground. No `run_in_background`, no `&`, no chained `sleep`.

### 5. Worktree Environment
External repo paths (specs, api-doc) are resolved via `.context-resolved.yaml`.
This file is auto-generated by `spec-drive setup` or can be created manually:
```bash
bash {specs_repo}/scripts/resolve-context.sh
```
Resolution order: env var -> local symlink -> git-worktree-discovery -> config hint -> prompt.
If `.context-resolved.yaml` exists, use `repos.specs` and `repos.api-doc` for absolute paths.
Legacy `specs` symlink still works as fallback.

## i18n Updates

### File Locations
- `src/locales/zh.json` (or `.ts`) -- Chinese (default)
- `src/locales/ja.json` -- Japanese
- `src/locales/en.json` -- English

### Rules
- No hardcoded user-visible strings.
- Add new strings to ALL locale files simultaneously.
- Interpolation variables must match across locales.
- Reference `specs/{version}/i18n/strings.md` as the source of truth.

## Spec-Driven Workflow

When operating under spec-drive orchestration:

1. **Read config**: `.claude/config.yaml` for paths and version.
2. **Check tasks**: `specs/{version}/tasks/web.md` for assignment and status.
3. **Collect context**: Feature YAML + Figma index + API doc + i18n strings.
4. **Lock task**: Update status `pending -> active` and commit.
5. **Implement**: Follow `ai/web.md` and `ai/ui.md` conventions.
6. **Build verify**: `{PACKAGE_MANAGER} run build && {PACKAGE_MANAGER} run typecheck` must pass.
7. **Update status**: `active -> done` and commit.
