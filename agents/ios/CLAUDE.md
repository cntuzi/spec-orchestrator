## Mandatory Rules (Claude Code only)

### 1. Build Verification

- **Foreground execution**: do not use `run_in_background`, `&`, or multiple `sleep` commands
- **Build command**: run the project build script after every change
  ```bash
  ./scripts/build.sh
  # Expected output: BUILD SUCCEEDED
  ```
- **Build loop**: modify -> build -> fix -> build again (never accumulate errors)

### 2. Worktree Environment

External repo paths (specs, api-doc) are resolved via `.context-resolved.yaml`.
This file is auto-generated by `spec-drive setup` or can be created manually:

```bash
bash {specs_repo}/scripts/resolve-context.sh
```

Resolution order: env var → local symlink → git-worktree-discovery → config hint → prompt.
If `.context-resolved.yaml` exists, use `repos.specs` and `repos.api-doc` for absolute paths.
Legacy `specs` symlink still works as fallback if `.context-resolved.yaml` is missing.

### 3. Change Scope

- Keep changes minimal and focused on the task at hand
- Do not refactor unrelated code incidentally
- Incremental migration limit: <= 15 files, <= 1500 lines per batch
- Delete/move files -> also clean up Xcode project references

## Spec Status Sync Protocol

When a task is completed, update the spec status files:

1. **Lock** (before starting): update `specs/{version}/tasks/{platform}.md` status from `pending` to `active`
2. **Complete** (after build succeeds): update status from `active` to `done`
3. **Commit**: include the status update in the task completion commit
4. **Blocked**: if build fails after 3 attempts, leave status as `active` and append blocking reason

Refer to `workflows/spec-protocol.md` and `docs/exec-protocol.md` in the spec-orchestrator repository for the full execution protocol.

## i18n Workflow

### Update Flow

1. Receive updated translation source (spreadsheet, CSV, or i18n service export)
2. Update the platform localization file (e.g., `Localizable.xcstrings`, `.strings`, `.stringsdict`)
3. Verify format specifiers match across all languages (`%@`, `%d`, `%f`)
4. Run build verification

### Common Issues

| Problem | Cause | Fix |
|---------|-------|-----|
| Format specifier mismatch | `%@` vs `%d` inconsistency | Unify to match the base language |
| Duplicate key | Same key used in multiple contexts | Check code to determine actual usage |
| Missing translation | Key not in localization file | Add the new key manually |
